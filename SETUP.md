# How to Create This GitHub Repo - Step by Step

## Step 1 - Create GitHub Account (if you don't have one)
1. Go to github.com
2. Click Sign Up
3. Use your professional email (arupd.jsr@gmail.com)
4. Choose username: arupdas-devops or arup-das-ocp (professional sounding)

## Step 2 - Create the Repository
1. Click the + icon top right on GitHub
2. Click "New repository"
3. Repository name: openshift-manifests-demo
4. Description: Production-grade Red Hat OpenShift 4.x manifests - RBAC, SCCs, Routes, NetworkPolicies, Workloads, and Operators
5. Set to PUBLIC (recruiters need to see it)
6. Check "Add a README file"
7. Click "Create repository"

## Step 3 - Upload Files Using GitHub Web Interface (Easiest)
1. Click "Add file" -> "Upload files"
2. Create folders by typing folder/filename in the text box
3. Upload each file one by one OR drag and drop

## Step 4 - OR Use Git Command Line (if Git is installed)
```bash
# Clone your new empty repo
git clone https://github.com/YOUR_USERNAME/openshift-manifests-demo.git
cd openshift-manifests-demo

# Copy all the files into this folder
# Then add, commit and push
git add .
git commit -m "Initial commit - OpenShift manifests for RBAC, SCCs, Routes, Networking, and Workloads"
git push origin main
```
